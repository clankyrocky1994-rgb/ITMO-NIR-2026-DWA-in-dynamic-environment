# DWA in Dynamic Environment for Mobile Manipulator

Проект посвящён локальной навигации мобильного манипулятора в динамической среде с использованием алгоритма **Dynamic Window Approach (DWA)**.
Симуляция выполнена в **MuJoCo**, движение мобильной базы реализовано через **mocap target** и обратную кинематику с использованием библиотеки **mink**.

В данной версии рассматривается только движение мобильной базы. Манипулятор зафиксирован в базовом положении и не участвует в захвате объектов. Основная задача робота — пройти по L-образному коридору, объезжая движущиеся цилиндрические препятствия.

---

## Цель проекта

Цель работы — реализовать и исследовать локальный планировщик движения мобильного робота в среде с динамическими препятствиями.

Робот должен:

* двигаться к заданной цели;
* проходить через промежуточные точки коридора;
* избегать столкновений со стенами;
* объезжать движущиеся цилиндрические препятствия;
* учитывать будущие положения препятствий;
* использовать аварийный режим при опасном сближении.

---

## Основная идея

В проекте используется связка:

```text
DWA → mocap target → mink IK → MuJoCo simulation
```

Алгоритм DWA не управляет суставами робота напрямую.
Он вычисляет допустимую скорость движения базы:

```text
vx, vy, omega
```

После этого в симуляции смещается `mocap`-тело `pinch_site_target`.
Библиотека `mink` решает задачу обратной кинематики и перемещает мобильную базу так, чтобы `pinch_site` робота следовал за этим target.

Таким образом, сохраняется старая логика управления через `mink`, но вместо движения по заранее построенному пути используется локальный DWA-планировщик.

## Описание алгоритма

На каждом шаге симуляции выполняются следующие действия:

1. Обновляются положения динамических препятствий.
2. Формируется список препятствий для DWA.
3. Определяется текущая локальная цель.
4. DWA перебирает допустимые скорости базы.
5. Для каждой скорости строится прогнозируемая траектория.
6. Траектория проверяется на столкновения со стенами и цилиндрами.
7. Выбирается команда с минимальной стоимостью.
8. По выбранной скорости смещается `mocap target`.
9. `mink` решает задачу обратной кинематики.
10. MuJoCo выполняет шаг симуляции.

Функция стоимости DWA учитывает:

* расстояние до цели;
* расстояние до препятствий;
* скорость движения;
* сглаженность команды;
* прогресс относительно текущей цели.

---

## Динамические препятствия

Препятствия представлены цилиндрами в MuJoCo-сцене.
Их положения изменяются во времени по заданному закону движения.

Для DWA каждое динамическое препятствие передаётся в формате:

```python
(x, y, radius, vx, vy)
```

Благодаря этому планировщик учитывает не только текущее положение цилиндра, но и его примерное будущее положение во время движения робота.

---

## Аварийный режим

Если динамическое препятствие оказывается слишком близко и движется в сторону робота, включается аварийный режим.

В этом режиме робот кратковременно откатывается назад относительно текущей цели.
Это позволяет избежать ситуации, когда робот уже начал объезд, но препятствие продолжает двигаться прямо на него.

После выхода из опасной зоны управление снова возвращается алгоритму DWA.
